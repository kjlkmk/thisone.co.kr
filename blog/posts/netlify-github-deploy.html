GitHub와 Netlify를 연동하여 정적 웹사이트 자동 배포하기

게시일: 2025년 12월 1일

< 목록으로 돌아가기

과거에는 웹사이트를 배포하기 위해 FTP 프로그램을 사용하여 매번 수동으로 파일을 서버에 업로드해야 했습니다. 하지만 이제 GitHub와 Netlify를 함께 사용하면, 내 컴퓨터에서 코드를 `push`하는 것만으로 웹사이트가 자동으로 업데이트되게 만들 수 있습니다. 이 글에서는 그 방법을 단계별로 알아보겠습니다.

이 방식을 'Git 기반 워크플로우'라고 부르며, 개발 과정을 매우 효율적이고 안정적으로 만들어주는 현대적인 웹 개발의 핵심입니다.

사전 준비물

  • 내 컴퓨터에 설치된 Git
  • GitHub 계정
  • Netlify 계정

1단계: 내 컴퓨터 프로젝트 폴더에서 Git 시작하기

먼저, 내 컴퓨터에 있는 웹사이트 프로젝트 폴더를 Git이 관리할 수 있도록 설정해야 합니다. 터미널을 열고 프로젝트 폴더로 이동한 후, 아래 명령어를 실행합니다.

# 1. Git 저장소로 초기화합니다.
git init

# 2. 모든 파일을 버전 관리에 추가합니다.
git add .

# 3. '첫 커밋'이라는 메시지와 함께 첫 버전을 저장합니다.
git commit -m "첫 커밋: 프로젝트 초기 버전"

이 과정은 프로젝트의 '첫 번째 스냅샷'을 찍는 것과 같습니다.

2단계: GitHub에서 원격 저장소 만들기

다음으로, 내 컴퓨터의 코드를 올려둘 GitHub 상의 온라인 저장 공간이 필요합니다. GitHub 웹사이트에 로그인하여 새로운 저장소(Repository)를 만듭니다. 이때, README 파일 없이 비어있는 상태로 만드는 것이 중요합니다.

3단계: 로컬 코드를 GitHub에 올리기(Push)

이제 내 컴퓨터(로컬)의 코드를 GitHub 저장소(원격)에 연결하고 업로드할 차례입니다. GitHub에서 저장소를 만들면 친절하게 안내해주는 명령어를 사용하면 됩니다.

# 1. 내 로컬 저장소에 원격 저장소 주소를 'origin'이라는 이름으로 등록합니다.
git remote add origin https://github.com/YourUsername/your-repo.git

# 2. 기본 브랜치 이름을 'main'으로 설정합니다.
git branch -M main

# 3. 코드를 'origin'의 'main' 브랜치로 업로드합니다.
git push -u origin main

이제 모든 코드가 GitHub에 안전하게 보관되었습니다.

4단계: Netlify와 GitHub 저장소 연결하기

마지막 단계입니다. Netlify가 GitHub의 코드 변경을 감지하고 자동으로 배포하도록 설정합니다.

  1. Netlify에 로그인하여 배포할 사이트를 선택합니다.
  2. Site configuration > Build & deploy 메뉴로 이동합니다.
  3. Link site to a Git repository 버튼을 눌러 방금 코드를 올린 GitHub 저장소를 선택합니다.
  4. 빌드 설정은 대부분 기본값을 그대로 사용하면 됩니다. (Build command와 Publish directory를 비워둡니다.)
  5. Deploy site 버튼을 누르면 첫 배포가 시작됩니다.

결론: 새로운 개발 워크플로우

이제 모든 설정이 끝났습니다. 앞으로의 작업 방식은 놀랍도록 간단해집니다.

  1. 내 컴퓨터에서 코드를 수정합니다.
  2. 터미널에서 git add ., git commit -m "수정 내용", git push 명령어를 실행합니다.
  3. 잠시 후, Netlify가 알아서 웹사이트에 변경사항을 반영합니다.

더 이상 수동 업로드의 번거로움 없이, 코드 관리에만 집중하며 효율적으로 웹사이트를 운영할 수 있게 되었습니다.
